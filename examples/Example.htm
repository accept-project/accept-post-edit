ACCEPT Post-Edit Plug-in

Use Case:

In this example, there are three paragraphs that will be used to demonstrate the Post-Edit functionality.

Clicking one of the segments triggers the process of opening the dialog box from which the content edition takes place.

Content Edition Placeholders:

Content to edit.
More content to edit.
Even more content to edit

Notes:

Replace [SOME POSTEDIT TASK -> TEXT ID] with valid text ID's. You need to create a Project and upload a task first.

Replace [ACCEPT_USER_NAME] with valid user name. You need to add a an user to the Post-Edit Project created above(or just use your own User Name - it works since it was the one creating the project).
